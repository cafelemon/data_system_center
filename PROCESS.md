## 已确认事项

- 一期目标：参照截图快速做出可运行、可演示、可修改的后台系统。
- 一期菜单：档案列表、用户管理、系统设置、数据统计。
- 业务边界：只管理纸质档案目录，不上传电子文件，不做文件预览，不做 OCR，不接知识库。
- 数据字段：先采用 PRD 中推荐字段，后续根据反馈通过数据库迁移和接口调整。
- 演示数据：P0 先冻结模拟数据口径，P1/P2 后逐步接入真实数据库和初始化数据脚本。
- UI 方向：以截图中的企业后台风格为准，强调简洁、克制、清晰、可演示。
- Git 边界：不复用父级 `/Users/jiafei/workspace` 仓库，当前项目目录需要单独初始化 Git。
