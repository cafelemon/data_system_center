# P1：工程底座搭建与基础联通

## 阶段目标

P1 目标是搭建正式工程底座，完成前端、后端、数据库、迁移、配置和基础联通能力，确保项目从第一天开始就能按接口和数据库继续迭代。

本阶段不实现登录、档案 CRUD、用户管理业务接口和统计业务接口；这些能力从 P2-P6 分阶段推进。

## 交付内容

- `backend/`：FastAPI 后端骨架。
- `frontend/`：Vue 3 + TypeScript + Vite 前端骨架。
- `docker-compose.yml`：PostgreSQL 本地开发数据库。
- `backend/alembic/`：Alembic 迁移配置和 P1 baseline migration。
- `backend/.env.example`：后端环境变量示例。
- `/api/health`：服务与数据库健康检查接口。
- `frontend/src/api/request.ts`：Axios 请求封装。
- `frontend/src/api/health.ts`：健康检查接口调用。
- `frontend/src/layouts/MainLayout.vue`：后台基础布局。
- `docs/references/screenshots/`：四个截图参考说明。

## 运行方式

### 1. 启动数据库

```bash
docker compose -p archive-system up -d postgres
```

### 2. 安装后端依赖

```bash
python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt
```

### 3. 执行数据库迁移

```bash
cd backend
../.venv/bin/alembic upgrade head
```

### 4. 启动后端

```bash
cd backend
../.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 18080 --reload
```

### 5. 安装并启动前端

```bash
cd frontend
npm install
npm run dev
```

访问 `http://127.0.0.1:15173`，前端会通过 Vite proxy 调用后端 `/api/health`。

本项目 PostgreSQL 使用本机端口 `15432` 映射容器内 `5432`，避免和机器上已有的 PostgreSQL 服务冲突。

## 验收标准

- 后端可启动，`GET /api/health` 返回 `code: 0`。
- PostgreSQL 可通过 SQLAlchemy 连接，健康接口中 `database` 为 `ok`。
- Alembic 可执行 `upgrade head`。
- 前端可启动并显示后台基础布局。
- 前端健康检查可以拿到后端返回结果。
