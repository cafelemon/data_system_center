# 双台账字段改造

## 阶段目标

在 P7 之后完成档案字段结构调整，将单一档案列表拆分为“纸质档案管理”和“电子档案管理”两个台账入口，同时让统计和导出按台账类型工作。

## 已实现范围

- `archives` 新增 `archive_medium`，支持 `paper` 和 `electronic`
- `archives` 新增 `internal_archive_type`，作为必填手工输入字段
- 历史档案统一迁移为纸质档案
- 新增 `paper_copies`、`archiver_name`、`electronic_storage_path`
- 原 `storage_location` 迁移为 `paper_storage_location`
- 纸质台账展示纸质份数和纸质存放位置
- 电子台账展示电子存储路径
- 归档部门字典强制切换为 15 个新部门
- 历史用户和历史档案引用统一迁移到“综合管理部”
- 旧部门停用保留，避免历史字典直接丢失
- 新增批量软删除接口 `DELETE /api/archives/batch`
- Excel 导出按纸质/电子台账输出不同字段，并包含内部档案类型
- 数据统计支持全部/纸质/电子筛选
- 数据统计继续按原档案类型字典分析，内部档案类型不进入统计分析

## 新归档部门

董事会办公室、综合管理部、北京公司、投资者关系部、财务与资产管理部、组织与人才发展中心、研发中心、生产中心、客户服务中心、数字与信息中心、质量体系部、临床与注册事务部、工程管理部、发展规划部、监察审计室。

## 接口变化

```text
GET /api/archives?archive_medium=paper
GET /api/archives?archive_medium=electronic
POST /api/archives
PUT /api/archives/{id}
DELETE /api/archives/{id}
DELETE /api/archives/batch
GET /api/archives/export?archive_medium=paper
GET /api/archives/export?archive_medium=electronic
GET /api/statistics/*?archive_medium=all|paper|electronic
```

## 字段说明

纸质档案表格字段：

```text
档案编号、档案名称、纸质份数、档案类型、内部档案类型、状态、保管期限、归档部门、归档人、归档日期、存放位置、责任人、操作
```

电子档案表格字段：

```text
档案编号、档案名称、档案类型、内部档案类型、状态、保管期限、归档部门、归档人、归档日期、存放路径、责任人、操作
```

## 权限规则

- 档案列表、详情、导出和统计需要登录。
- 新建、编辑、单条删除和批量删除需要管理员角色。
- 普通用户不显示删除入口。
- 批量删除仍为软删除，只写入 `deleted_at`。

## 验收结果

- Alembic 升级成功，历史档案 `archive_medium=paper`。
- 旧 `storage_location` 已迁移到 `paper_storage_location`。
- 新 15 个归档部门启用，旧部门停用。
- 纸质/电子列表筛选、详情、新建、编辑、单删、批量删除和导出可用。
- 统计接口在 `all`、`paper`、`electronic` 三种口径下返回成功。
- 前端生产构建通过。
