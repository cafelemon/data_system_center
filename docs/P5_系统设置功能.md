# P5 系统设置功能

## 阶段目标

P5 完成系统设置页面的真实功能，优先实现档案类型配置和保管期限配置，让基础字典可以通过后台维护。

本阶段不实现档案状态、部门、角色等其他字典维护。

## 已实现范围

- 档案类型列表接口：`GET /api/settings/archive-types`
- 档案类型新增接口：`POST /api/settings/archive-types`
- 档案类型编辑接口：`PUT /api/settings/archive-types/{archive_type_id}`
- 档案类型启用/禁用接口：`PATCH /api/settings/archive-types/{archive_type_id}/status`
- 档案类型删除接口：`DELETE /api/settings/archive-types/{archive_type_id}`
- 保管期限列表接口：`GET /api/settings/retention-periods`
- 保管期限新增接口：`POST /api/settings/retention-periods`
- 保管期限编辑接口：`PUT /api/settings/retention-periods/{retention_period_id}`
- 保管期限启用/禁用接口：`PATCH /api/settings/retention-periods/{retention_period_id}/status`
- 保管期限删除接口：`DELETE /api/settings/retention-periods/{retention_period_id}`
- 前端系统设置页签、表格、弹窗、状态开关和删除校验提示
- 档案新建表单只展示启用的档案类型和保管期限

## 权限规则

- 所有系统设置接口都需要管理员角色。
- 删除采用物理删除，但删除前会校验是否被档案引用。
- 被任何档案引用的档案类型或保管期限不能删除，包括软删除档案仍引用的配置。

## 数据规则

- 档案类型：`name`、`code` 必填且唯一。
- 保管期限：`name` 必填且唯一，`years` 允许 `0`、正数和 `-1`。
- 新增配置未传 `sort_order` 时，自动按当前最大排序值加 10。
- 启用/禁用只更新 `enabled`。
- 新增、编辑、启用/禁用、删除会写入基础操作日志。

## 档案表单联动

- `/api/archives/options` 继续返回配置的 `enabled` 状态。
- 新建档案时，前端只展示启用的档案类型和保管期限。
- 编辑档案时，前端展示启用项，并保留当前档案已选中的停用项。
- 后端新增档案时禁止选择停用档案类型或停用保管期限。
- 后端编辑档案时允许保持原本已停用的历史值，但禁止切换到其他停用项。

## 验收结果

- 后端 Python 编译检查通过。
- Alembic 当前 head 可升级。
- 无 Token 访问系统设置接口返回 401。
- 普通用户访问系统设置接口返回 403。
- admin 可读取档案类型和保管期限列表。
- 临时档案类型可新增、编辑、启用/禁用和删除。
- 临时保管期限可新增、编辑、启用/禁用和删除。
- 被档案引用的配置删除会返回 409。
- 停用配置后，新建档案不可再选择该配置。
- 前端生产构建通过。
