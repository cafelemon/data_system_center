# P0：项目启动与目标冻结

## 项目目标说明

纸质档案目录管理系统一期目标是参照已确认截图，快速落地一套可运行、可演示、可修改的企业内部后台系统。

当前阶段不追求一次性完成最终版业务细则，而是先冻结一期建设方向，让后续 P1-P9 能围绕同一个目标持续推进。系统从第一版开始按正式工程建设，不做脱离后端和数据库的纯静态展示页。

系统边界如下：

- 只管理纸质档案目录。
- 不上传电子文件。
- 不做文件预览。
- 不做 OCR。
- 不接知识库。
- 字段先采用推荐版本，后续根据领导和信息中心反馈继续调整。

## 一期功能范围

一期核心菜单固定为四个：

1. 档案列表
2. 用户管理
3. 系统设置
4. 数据统计

### 档案列表

用于管理所有纸质档案目录记录，支持关键词搜索、档案类型筛选、所属部门筛选、状态筛选、重置筛选、查看、编辑、新建和导出 Excel。

首版列表字段：

- 档案编号
- 档案名称
- 档案类型
- 状态
- 保管期限
- 所属部门
- 归档日期
- 存放位置
- 责任人
- 操作

### 用户管理

用于管理系统账号、角色、部门和启用状态。首版支持用户列表、关键词搜索、角色筛选、状态筛选、部门筛选、编辑和禁用/删除操作。

### 系统设置

用于维护基础字典，首版优先实现：

- 档案类型配置
- 保管期限配置

页面需要支持页签切换、新增配置、编辑配置、启用/禁用和删除校验。

### 数据统计

用于展示档案资产概况与分布趋势。首版包含：

- 顶部指标卡片：档案总数、本月新增、即将到期、涉及部门
- 档案状态分布
- 档案类型分布环形图
- 部门档案数量柱状图
- 月度入库趋势折线图

## 技术栈说明

前端采用 Vue 3 + TypeScript + Vite + Element Plus + ECharts。Element Plus 用于快速搭建表格、筛选、表单、弹窗、分页等企业后台组件；ECharts 用于统计页图表；Pinia、Vue Router 和 Axios 分别负责状态、路由和接口请求。

后端采用 FastAPI + SQLAlchemy ORM + Pydantic + Alembic。FastAPI 用于快速提供接口和 Swagger 文档；SQLAlchemy 管理数据模型和查询；Pydantic 负责接口入参与出参校验；Alembic 管理数据库迁移。

数据库采用 PostgreSQL，首版支持万级到十万级纸质档案目录数据的分页查询、筛选和统计。认证与权限采用 JWT + RBAC，导入导出能力使用 openpyxl。部署目标采用 Docker Compose + Nginx + Ubuntu Server。

## 第一版开发任务清单

### P1：工程底座搭建与基础联通

- 初始化 FastAPI 后端项目。
- 初始化 Vue 3 + TypeScript + Vite 前端项目。
- 配置 PostgreSQL、SQLAlchemy 和 Alembic。
- 建立统一响应格式、异常处理和日志模块。
- 建立 Axios 请求封装、前端路由和基础 Layout。
- 完成 `/api/health` 前后端联通。

### P2：核心数据模型与初始化数据

- 建立 users、roles、departments、archives、archive_types、archive_statuses、retention_periods、operation_logs 等核心表。
- 编写初始 Alembic 迁移。
- 初始化角色、部门、档案类型、档案状态、保管期限、管理员账号和档案测试数据。

### P3：登录、用户与权限基础功能

- 实现登录、JWT、当前用户信息接口。
- 实现用户列表、搜索、筛选、新增、编辑、启用/禁用。
- 前端接入登录和用户管理真实接口。

### P4：档案列表核心功能

- 实现档案分页列表、关键词搜索、多条件筛选、详情、新增、编辑和软删除预留接口。
- 前端完成档案列表、筛选、分页、新建、编辑和详情弹窗联调。

### P5：系统设置功能

- 实现档案类型和保管期限的列表、新增、编辑、启用/禁用和删除校验。
- 前端完成系统设置页签、配置弹窗和开关联调。

### P6：数据统计看板

- 实现统计概览、状态分布、类型分布、部门排行和月度趋势接口。
- 前端接入指标卡片、状态分布、环形图、柱状图和折线图。

### P7：Excel 导出与基础操作日志

- 实现按筛选条件导出档案列表 Excel。
- 记录登录、新增档案、编辑档案、导出和系统配置修改日志。

### P8：部署基础

- 编写 Dockerfile、docker-compose 和 Nginx 配置。
- 完成本地容器化启动和部署说明。

### P9：测试、演示数据与交付说明

- 完成核心功能测试、不同分辨率页面检查、演示数据说明和交付文档。

## P0 验收标准

- 项目可以直接进入 P1 开发。
- 一期目标、菜单、业务边界、技术栈和开发任务清单已经冻结。
- 所有后续开发围绕截图中的四个页面推进。
- 当前目录建立独立 Git 仓库，不复用父级目录 Git。
- 新增开发进度文档，后续阶段持续维护。
