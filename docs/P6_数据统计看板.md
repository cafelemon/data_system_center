# P6 数据统计看板

## 阶段目标

P6 完成数据统计看板的真实联通，让“数据统计”页面展示数据库中的档案概览、状态分布、类型分布、部门排行和月度趋势。

本阶段同时将本机开发演示调整为局域网可访问：同网段设备可通过 `http://<本机局域网IP>:15173` 打开系统并登录。

## 已实现范围

- 统计概览接口：`GET /api/statistics/overview`
- 状态分布接口：`GET /api/statistics/status-distribution`
- 类型分布接口：`GET /api/statistics/type-distribution`
- 部门排行接口：`GET /api/statistics/department-ranking?limit=8`
- 月度趋势接口：`GET /api/statistics/monthly-trend?months=12`
- 前端数据统计页面真实接口联调
- 顶部指标卡片、状态分布卡片、环形图、柱状图和折线图
- 前端开发与预览服务绑定 `0.0.0.0`
- 后端启动文档调整为绑定 `0.0.0.0`

## 权限规则

- 所有统计接口都需要登录。
- 统计接口不要求管理员权限，普通用户也可以查看统计看板。

## 统计口径

- 所有统计只计算未软删除档案，即 `deleted_at IS NULL`。
- 档案总数：未删除档案总数。
- 本月新增：`created_at` 落在当前自然月的档案数。
- 即将到期：`archive_date + retention_period.years` 落在今天起未来 90 天内的档案数；`永久` 和 `其他` 不计入。
- 涉及部门：有档案记录的去重部门数量。
- 状态分布：返回全部默认档案状态，按状态 `sort_order` 排序，百分比以未删除档案总数为分母。
- 类型分布：返回全部档案类型，按类型 `sort_order` 排序。
- 部门排行：按档案数量倒序返回前 8 个部门。
- 月度趋势：按 `archive_date` 月份统计，默认返回以数据库最大归档日期所在月为终点的最近 12 个月。

## 接口约定

概览响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total_archives": 120,
    "monthly_new": 0,
    "expiring_soon": 0,
    "department_count": 11
  }
}
```

分布响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": 1,
        "name": "在库",
        "code": "ZK",
        "count": 20,
        "percentage": 16.7
      }
    ],
    "total": 120
  }
}
```

## 局域网演示

后端启动：

```bash
cd backend
../.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 18080 --reload
```

前端启动：

```bash
cd frontend
npm run dev
```

同网段设备访问：

```text
http://<本机局域网IP>:15173
```

局域网演示应只在可信网络中使用。演示账号为 `admin / Admin@123456`。
