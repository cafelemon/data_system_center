# P4 档案列表核心功能

## 阶段目标

P4 完成档案列表核心业务闭环，让首页菜单中的“档案列表”从占位页变成真实数据库页面，支持分页、搜索、筛选、查看详情、新建和编辑。后续双台账改造已将该入口拆分为“纸质档案管理”和“电子档案管理”。

本阶段仍不实现 Excel 导出，页面保留导出按钮位置，真实导出进入后续阶段。

## 已实现范围

- 档案分页列表接口：`GET /api/archives`
- 档案详情接口：`GET /api/archives/{archive_id}`
- 档案新增接口：`POST /api/archives`
- 档案编辑接口：`PUT /api/archives/{archive_id}`
- 档案软删除接口：`DELETE /api/archives/{archive_id}`
- 档案筛选选项接口：`GET /api/archives/options`
- 列表关键词搜索：档案编号、档案名称
- 列表筛选：内部档案类型、归档部门、状态、台账类型
- 前端档案列表真实接口联调
- 前端新建、编辑、查看详情弹窗
- 状态标签颜色映射

## 权限规则

- 档案列表、详情和筛选选项需要登录。
- 档案新增、编辑和软删除需要管理员角色。
- 软删除只写入 `deleted_at`，列表默认不展示已删除档案。

## 字段范围

P4 前端表格展示：

- 档案编号
- 档案名称
- 档案类型
- 内部档案类型
- 状态
- 保管期限
- 纸质份数
- 归档部门
- 归档人
- 归档日期
- 存放位置
- 责任人

新增、编辑和详情弹窗包含：

- 档案编号、档案名称、档案类型、内部档案类型、状态、保管期限、归档部门
- 归档人、归档日期、纸质存放位置或电子存储路径、责任人、档案年度
- 密级、重要程度、项目、关联单位、合同编号、关键词、备注

## 接口约定

列表响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [],
    "total": 120,
    "page": 1,
    "page_size": 10
  }
}
```

档案对象包含：

```json
{
  "id": 1,
  "archive_no": "DA-2020-0001",
  "title": "2024年度财务报表",
  "archive_type": {},
  "internal_archive_type": "文书档案",
  "status": {},
  "retention_period": {},
  "department": {},
  "archive_medium": "paper",
  "paper_copies": 1,
  "archive_date": "2020-01-15",
  "paper_storage_location": "A区-01柜",
  "electronic_storage_path": null,
  "archiver_name": "李四",
  "owner_name": "张三",
  "archive_year": 2020
}
```

## 验收结果

- 后端 Python 编译检查通过。
- Alembic 当前 head 可升级。
- admin 登录成功。
- 无 Token 访问档案列表返回 401。
- 带 Token 可读取档案列表和详情。
- 关键词、内部档案类型、归档部门、状态、台账类型筛选可用。
- 新增、编辑、软删除临时测试档案 API 可用。
- 前端生产构建通过。
