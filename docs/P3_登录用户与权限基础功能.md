# P3 登录用户与权限基础功能

## 阶段目标

P3 完成登录认证和用户管理的真实联通，让系统从工程底座进入可登录、可进入后台、可维护用户账号的状态。

本阶段仍不实现档案 CRUD、系统设置字典维护和统计接口，这些功能从 P4 起继续推进。

## 已实现范围

- 登录接口：`POST /api/auth/login`
- 当前用户接口：`GET /api/auth/me`
- JWT Token 生成、校验和过期时间配置
- 用户列表接口：`GET /api/users`
- 用户筛选：关键词、角色、状态、部门
- 用户新增接口：`POST /api/users`
- 用户编辑接口：`PUT /api/users/{user_id}`
- 用户启用/禁用接口：`PATCH /api/users/{user_id}/status`
- 用户管理筛选选项接口：`GET /api/users/options`
- 前端登录页、Token 本地保存、Axios Bearer Token 注入
- 前端路由登录守卫
- 前端用户管理页面接入真实数据库

## 权限规则

- `/api/auth/login` 不需要登录。
- `/api/auth/me` 需要有效 Bearer Token。
- `/api/users` 系列接口需要管理员角色。
- 禁用用户不能登录。
- 当前登录用户不能禁用自己的账号。

## 默认账号

- 管理员：`admin / Admin@123456`
- 普通演示用户默认密码：`User@123456`

默认密码来自 P2 初始化脚本和环境变量：

- `DEFAULT_ADMIN_PASSWORD`
- `DEFAULT_USER_PASSWORD`

## 配置项

后端 `.env.example` 已新增：

```env
JWT_SECRET_KEY=please-change-this-development-secret
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=480
```

本地开发可直接使用示例值；部署或正式演示环境需要替换 `JWT_SECRET_KEY`。

## 接口约定

登录响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "access_token": "jwt-token",
    "token_type": "bearer",
    "expires_in": 28800,
    "user": {}
  }
}
```

用户列表响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [],
    "total": 8,
    "page": 1,
    "page_size": 10
  }
}
```

## 验收结果

- 后端 Python 编译检查通过。
- Alembic 当前 head 可升级。
- admin 登录成功并返回 JWT。
- 禁用用户登录失败。
- 无 Token 访问受保护接口返回 401。
- 管理员 Token 可访问当前用户和用户列表。
- 用户列表关键词、角色、状态、部门筛选可用。
- 新增、编辑、启用/禁用用户 API 可用。
- 前端生产构建通过。
