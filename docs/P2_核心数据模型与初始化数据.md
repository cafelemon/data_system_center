# P2：核心数据模型与初始化数据

## 阶段目标

P2 完成系统核心数据表、基础字典、初始化账号和档案演示数据，为 P3-P6 的登录、用户管理、档案列表、系统设置和数据统计提供真实数据库基础。

本阶段不新增 HTTP API，不做前端业务页面联调。

## 交付内容

- SQLAlchemy ORM 模型：用户、角色、部门、档案主表、档案类型、档案状态、保管期限、操作日志。
- Alembic 迁移：`20260505_0002_p2_core_schema.py`。
- 幂等初始化脚本：`backend/scripts/seed_data.py`。
- 默认角色、部门、档案类型、档案状态、保管期限。
- 默认演示用户：admin、张三、李四、王五、赵六、钱七。
- 不少于 120 条档案测试数据。
- 四张截图 PNG 参考资产。

## 初始化命令

```bash
cd backend
../.venv/bin/alembic upgrade head
../.venv/bin/python scripts/seed_data.py
```

默认密码可通过环境变量覆盖：

```bash
DEFAULT_ADMIN_PASSWORD=Admin@123456
DEFAULT_USER_PASSWORD=User@123456
```

## 默认数据说明

- 管理员账号：`admin / Admin@123456`
- 普通演示用户默认密码：`User@123456`
- 档案测试数据只用于联调和演示，后续可清理或用正式导入数据替换。
- `人事档案` 按截图设置为禁用，但保留历史测试档案引用能力。

## 验收标准

- P2 迁移可成功执行。
- 初始化脚本可重复执行，不产生重复字典、用户和档案。
- 数据库中角色不少于 3 条、部门不少于 10 条、用户不少于 6 条、档案类型不少于 8 条、档案状态 6 条、保管期限不少于 5 条、档案不少于 120 条。
