# 纸质档案目录管理系统落地方案

## 0. 当前推进口径

先参照现有截图样式，快速做出一套可运行、可演示、可修改的纸质档案目录管理系统。

本阶段目标不是一次性做成最终版，而是先把截图中已经明确的页面和功能完整落地，包括：档案列表、用户管理、系统设置、数据统计四个核心模块。后续领导或信息中心提出字段、权限、流程、统计口径等要求后，再基于已有系统继续调整。

本系统当前只管理纸质档案目录，不上传电子文件，不做文件预览，不做OCR，不接知识库。系统核心是对纸质档案的编号、名称、类型、状态、保管期限、所属部门、归档日期、存放位置、责任人等信息进行电子化台账管理。

# 一、技术栈方案

## 1.1 总体技术架构

```text
用户浏览器
   ↓
前端管理后台：Vue 3 + TypeScript + Element Plus + ECharts
   ↓
后端接口服务：FastAPI + SQLAlchemy ORM + Pydantic + Alembic
   ↓
业务数据库：PostgreSQL
   ↓
部署运行：Docker Compose + Nginx + Ubuntu Server
```

## 1.2 技术选型说明

| 层级 | 技术选型 | 说明 |
|---|---|---|
| 前端框架 | Vue 3 | 适合快速搭建企业后台系统 |
| 开发语言 | TypeScript | 提高代码稳定性和可维护性 |
| UI组件库 | Element Plus | 表格、筛选、表单、弹窗、分页组件成熟，适合复刻截图样式 |
| 图表库 | ECharts | 用于数据统计页的环形图、柱状图、趋势图 |
| 状态管理 | Pinia | 管理登录用户、权限、菜单状态 |
| 路由管理 | Vue Router | 管理左侧菜单页面切换 |
| 请求工具 | Axios | 前后端接口请求 |
| 后端框架 | FastAPI | 接口开发快，自动生成Swagger文档 |
| ORM | SQLAlchemy ORM | 管理数据库模型和查询 |
| 数据校验 | Pydantic | 校验接口入参和返回结构 |
| 数据迁移 | Alembic | 管理数据库表结构变更 |
| 数据库 | PostgreSQL | 支持万级、十万级目录数据，查询和统计稳定 |
| 导入导出 | openpyxl | 支持Excel模板、导入、导出 |
| 权限认证 | JWT + RBAC | 实现登录、角色、菜单和按钮权限 |
| 部署 | Docker Compose + Nginx | 便于内网服务器部署和后续迁移 |

## 1.3 开发与部署方式

### 开发阶段

```text
Mac 本地开发
前端：Node.js + Vite 本地启动
后端：Python 虚拟环境 + FastAPI 本地启动
数据库：优先使用 Docker PostgreSQL
```

### 正式部署阶段

```text
Ubuntu Server
Docker Compose 编排
Nginx 反向代理
PostgreSQL 数据持久化
定期数据库备份
```

# 二、PRD 产品需求文档

## 2.1 产品名称

纸质档案目录管理系统

英文暂定：Archive Directory System

## 2.2 产品定位

本系统用于公司内部纸质档案目录的电子化管理，通过统一的后台系统管理纸质档案台账，实现档案目录查询、录入、编辑、筛选、统计、用户管理和基础配置。

系统一期以“先做出来、可演示、可修改”为核心原则，优先复刻截图中的后台系统能力，不等待完整业务细则。

## 2.3 一期建设目标

1. 直接按照正式工程底座推进，不做脱离后端和数据库的静态展示页；
2. 从第一阶段开始同步搭建前端、后端、数据库、接口、权限和迁移体系；
3. 以截图样式作为界面目标，但所有页面必须逐步接入真实接口和数据库；
4. 实现档案列表的查询、筛选、新增、编辑、查看和导出；
5. 实现用户管理、角色权限、系统设置和数据统计；
6. 预置一批初始化数据用于联调和演示，但不采用长期 mock 数据方案；
7. 数据字段先采用推荐版本，后续根据领导和信息中心意见通过数据库迁移和接口调整继续迭代。

## 2.4 用户角色

| 用户角色 | 使用目标 |
|---|---|
| 系统管理员 | 管理用户、配置字典、查看统计、维护全部档案目录 |
| 档案管理员 | 新增、编辑、查询、导出档案目录 |
| 普通用户 | 查询可见范围内的档案目录 |
| 管理层用户 | 查看档案列表和统计数据，原则上不参与维护 |

# 十三、最终结论

当前最合适的推进方式是：不等待完整需求，但也不做纯静态展示品，而是直接按照正式工程底座推进。

第一版系统仍然围绕截图中的四个页面：

```text
档案列表
用户管理
系统设置
数据统计
```

但实现路径应调整为：

```text
先搭工程底座
先建数据库模型
先做真实接口
再做页面联调
最后按截图样式打磨界面
```

第一版系统重点能力是：

```text
登录认证
档案查询
档案筛选
新增档案
编辑档案
查看档案
导出Excel
用户管理
字典配置
统计展示
基础日志
```

技术上采用：

```text
Vue 3 + Element Plus + ECharts
FastAPI + SQLAlchemy + PostgreSQL
Alembic + JWT + RBAC
Docker Compose + Nginx
```

这样可以避免系统沦为一次性展示页面，也能保证后续领导或信息中心提出字段、权限、流程、统计口径等要求时，可以在真实工程上继续迭代。
